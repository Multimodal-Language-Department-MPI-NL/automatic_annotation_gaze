# 👁️ GroundingGaze: An Interactive Pipeline for Automatic Gaze Annotation

A Gradio-based web application for video object tracking using Grounded SAM2, with integrated gaze data processing for Tobii Pro 3 and Pupil Neon eye trackers.

Overview
This tool combines the power of Grounded SAM2 (Segment Anything Model 2 with grounding) with eye-tracking data to enable automated, gaze-aware object segmentation and tracking across multiple videos. It is particularly suited for research workflows where you want to track objects of interest identified through participant gaze.
---

## Features

- 🎯 **Grounded SAM2 tracking** — automatic object detection and segmentation using text prompts or gaze-guided inputs
- 👁️ **Gaze integration** — supports Tobii Pro 3 and Pupil Neon eye tracker data formats
- 🗂️ **Multi-video processing** — batch process multiple video files in one session
- 🖥️ **Gradio web UI** — browser-based interface, no coding required to run
- 📤 **CSV export** — converts raw gaze data to a unified CSV format for downstream analysis

## Requirements

- Python 3.9+
- CUDA-compatible GPU (strongly recommended for SAM2 inference)
- [Grounded SAM2](https://github.com/IDEA-Research/Grounded-SAM-2) and its dependencies installed
- Gradio

Install Python dependencies:

> **Note:** Follow the [Grounded SAM2 installation guide](https://github.com/IDEA-Research/Grounded-SAM-2#installation) to set up model weights and CUDA dependencies before running.

---

### Model Checkpoints
 
Download the required model weights and place them in the correct directories before running.
 
**Grounding DINO** — place checkpoints in `gdino_checkpoints/`:
 
```bash
mkdir -p gdino_checkpoints
# Download from the Grounded SAM2 repo, e.g.:
# gdino_checkpoints/groundingdino_swint_ogc.pth
```
 
**SAM2** — place checkpoints in `checkpoints/`:
 
```bash
mkdir -p checkpoints
# Download from the SAM2 repo, e.g.:
# checkpoints/sam2.1_hiera_large.pt
```

## Usage

Start the application:

```bash
python main.py
```

The app will be available at:
- **Local:** `http://localhost:7870`
- **Public share link** (auto-generated by Gradio): printed in the terminal on startup

### Workflow

1. Upload one or more video files via the Gradio interface
2. Optionally upload gaze data (Tobii Pro 3 or Pupil Neon format)
3. Configure tracking parameters (text prompt or gaze-guided mode)
4. Run the tracker and review segmentation results
5. Export results or processed gaze CSV as needed

---

## Gaze Data Support

| Device | Format | Notes |
|---|---|---|
| Tobii Pro 3 | Raw Tobii export | Converted to CSV via `TobiiPro3_PupilsNeon_to_csv` |
| Pupil Neon | Raw Pupil export | Converted to CSV via `TobiiPro3_PupilsNeon_to_csv` |

The `GazeProcessor` aligns gaze coordinates to video frames and passes fixation points to the tracker for region-of-interest guided segmentation.

## Configuration

The server launch settings in `main.py` can be adjusted:

```python
demo.launch(
    server_name="0.0.0.0",   # Listen on all interfaces
    server_port=7870,          # Change port here
    share=True,                # Set to False to disable public link
    debug=True
)
```
